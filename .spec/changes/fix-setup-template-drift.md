# Fix setup template drift

## What
Align the template generated by `setup.sh` with the live `.spec/changes/_template.md` so downstream repos start with the current workflow contract, including the `files:` metadata line used by the scope gate.

## Acceptance criteria
- [ ] `setup.sh` generates a change template that includes the same top metadata shape as the live template, including a blank `files:` line below `status:`
- [ ] The generated template still preserves the current closure prompts and section order
- [ ] Repo-local verification covers the scaffolded template shape via a concrete shell assertion in `tests/test-spec-gate.sh` or another existing repo-local test path

## Notes
- Keep the setup template intentionally simple; this change is about alignment, not redesign
- Updating already-existing downstream `_template.md` files is out of scope; this change fixes new scaffolds generated by `setup.sh`
