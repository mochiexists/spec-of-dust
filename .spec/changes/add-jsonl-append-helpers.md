# Add JSONL append helpers

## What
Add bash scripts that safely append structured JSONL entries to `devlog.jsonl` and `flowlog.jsonl`, so agents call a command with named args instead of hand-crafting JSON. This eliminates format drift and keeps the logs machine-readable.

## Acceptance criteria
- [ ] `scripts/devlog.sh` accepts named args (`--kind`, `--summary`, `--reason`, `--file`) and appends a valid JSONL line to `.spec/devlog.jsonl` with `ts`, `event`, `kind`, `summary`, `reason`, `files`, and `command` fields
- [ ] `scripts/flowlog.sh` accepts named args (`--change`, `--agent`, `--sentiment`, `--divergence`, `--friction`, `--suggestion`) and appends a valid JSONL line to `.spec/flowlog.jsonl` with `ts`, `agent`, `change`, `flow_divergence`, `friction`, `suggestion`, and `sentiment` fields
- [ ] Both scripts validate required fields and enum values (`--kind` must be one of `typo|version-bump|comment|config-tweak|one-line-fix`; `--sentiment` must be `smooth|rough|blocked`) and exit non-zero with a usage message on invalid input
- [ ] Both scripts escape `"`, `\`, and control characters in string values to produce valid JSON, or reject inputs that cannot be safely escaped
- [ ] `.spec/FLOW.md` mentions the helper scripts as the preferred way to append entries

## Notes
- Zero dependencies — bash and standard Unix tools only
- `ts` is auto-generated by the script (ISO 8601 UTC), not passed by the caller
- `devlog.sh` auto-sets `event: "skip-no-verify"` and `command: "git commit --no-verify"` since those are always the same
- `--file` for devlog takes a single file path (skip mode only allows one target file); output as a one-element JSON array for schema consistency
- Free-text fields default to empty string if not provided
- Required fields: devlog needs `--kind`, `--summary`, `--reason`, `--file`; flowlog needs `--change`, `--agent`, `--sentiment`
